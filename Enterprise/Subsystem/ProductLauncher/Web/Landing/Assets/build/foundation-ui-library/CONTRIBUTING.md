# Contributing to the Foundation UI Library

The following is a set of guidelines to contribute to this library. These are
guidelines and not rules, so use your best judgement and submit recommendations
via a pull request at any time.

In order to make changes to this library, you will need to 
- Clone this repo
- Create a new branch (e.g. `calendar-styling-updates`)
- Make your changes and commit them to the new branch
- Push your new branch to GitLab
- Create a pull request for your new branch into master

## Deployments
Assets in the `dist` directory are automatically deployed to the Realpage CDN.  Simply create a new release in GitHub with a [semantically versioned tag](http://semver.org) (e.g. - vX.X.X) and the new deploy will be available within a minute or two.  Be sure to include a list of fixes and breaking/non-breaking changes.
